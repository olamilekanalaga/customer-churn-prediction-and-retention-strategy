# GitHub Screenshot Guidance

Use screenshots in the README because recruiters scan visually before reading deeply.

Recommended screenshots:

1. Dashboard top section showing KPIs and churn risk score.
2. Business insights tab showing churn by contract and tenure patterns.
3. Model performance tab showing model comparison and ROC curve.

Suggested GitHub order:

1. One dashboard screenshot near the top of the README.
2. One model-driver chart in the results section.
3. One business chart in the insights section.

Keep screenshots clean and cropped around the dashboard content. Avoid including the browser address bar if possible.
